这个 `biomni/biorxiv_scripts` 文件夹里只有 3 个脚本，它们合起来实现的是一条很完整的流程：

**从 bioRxiv 元数据里选论文 → 下载并解析 PDF → 用 LLM 从论文中抽取“常见计算任务 / 数据库 / 软件” → 汇总统计 → 再把任务描述生成成 Python 脚本。** 目录页显示这 3 个文件分别是 `extract_biorxiv_tasks.py`、`generate_function.py` 和 `process_all_subjects.py`。([GitHub][1])

下面按文件分别说明。

## 1. `extract_biorxiv_tasks.py`

这是这个目录里最核心的“单学科处理脚本”。它的作用是：
**针对某一个 bioRxiv 学科方向，从元数据 CSV 里选论文，下载 PDF，抽文本，调用 `PaperTaskExtractor` 提取论文中的常见任务、数据库和软件，然后输出单篇结果和汇总文件。** ([GitHub][2])

它主要做了几件事：

* 解析命令行参数，支持指定 `--subject`、`--limit`、`--metadata-path`、`--output-dir`、`--model`、`--chunk-size`、`--chunk-overlap`、`--max-paper-length`、`--save-pdfs`、`--random-sample`。这说明它可以灵活控制处理哪个学科、抽多少篇、用哪个模型和文本切块参数。([GitHub][2])
* 通过 `load_papers_from_csv(...)` 读取 bioRxiv metadata CSV，并筛选 `published != "NA"` 的论文；如果 `subject == "all"`，就取所有已发表论文，否则按 `category` 精确匹配学科。数量超过 `limit` 时，可以 `head(limit)` 或随机抽样。([GitHub][2])
* 通过 `download_pdf(...)` 按 DOI 拼出 bioRxiv PDF 地址 `https://www.biorxiv.org/content/{doi}v1.full.pdf`，下载 PDF，并调用 `extract_text_from_pdf(...)` 用 `PyPDF2.PdfReader` 提取全文文本。为了避免限流，还会加 `1–3` 秒的随机延迟。([GitHub][2])
* 通过 `truncate_text(...)` 在论文文本过长时按句子边界截断到 `max-paper-length`，尽量避免把句子截断在中间。([GitHub][2])
* 在 `process_papers(...)` 里初始化 `PaperTaskExtractor(llm=..., chunk_size=..., chunk_overlap=...)`，然后对每篇论文执行 `extractor.go(pdf_text)`，把返回结果和论文元数据一起保存为 JSON。输出里包含论文 DOI、标题、作者、摘要、日期、学科和文本长度，以及提取出的 `tasks`、`databases`、`software`。([GitHub][2])
* 在 `generate_summary(...)` 里把所有论文的任务、数据库、软件合并，分别输出 `tasks_summary.csv`、`databases_summary.csv`、`software_summary.csv`，再生成一个 `frequency_summary.json` 做频次统计。([GitHub][2])

所以这个脚本的本质角色是：

**“单学科论文挖掘器”**
负责从论文中抽取可复用的方法学元素，而不是直接做生物分析。这个功能和它调用的 `PaperTaskExtractor` 也一致。([GitHub][2])

## 2. `generate_function.py`

这个脚本是第二步：
**把前面得到的“任务描述”进一步变成 Python 脚本。** 它是一个命令行工具，直接调用 `biomni.agent.function_generator.FunctionGenerator`。([GitHub][3])

它的主要逻辑很简单：

* 通过参数 `--task / -t` 指定一个 JSON 文件，里面应包含任务列表。([GitHub][3])
* 通过 `json.load(...)` 读取这个文件，并取出 `task_list["tasks"]` 作为任务描述列表。([GitHub][3])
* 初始化 `FunctionGenerator(llm=args.model, temperature=args.temperature)`。默认模型是 `claude-3-7-sonnet-latest`，默认温度是 `0.7`。([GitHub][3])
* 对每个任务描述调用 `function_generator.go(desc)`，得到：

  * 生成的脚本文件名
  * 生成的 Python 代码文本 ([GitHub][3])
* 把每个生成结果写入 `output-dir` 指定的目录中，默认目录名是 `generated_functions`。([GitHub][3])

所以它的定位是：

**“任务描述 → Python 函数/脚本”的批量代码生成器。**
它本身不负责抽取任务，只负责把抽取好的任务文本转成代码。([GitHub][3])

## 3. `process_all_subjects.py`

这个脚本是总控批处理器。它的作用是：
**遍历 bioRxiv 的多个学科，对每个学科调用 `extract_biorxiv_tasks.py`，最后把所有学科的频次结果合并成总汇总。** ([GitHub][4])

它实现了几层功能：

* 先在代码里定义了一个 `BIORXIV_SUBJECTS` 列表，包含 `neuroscience`、`bioinformatics`、`genomics`、`systems biology`、`immunology`、`synthetic biology`、`pathology` 等一整组 bioRxiv 学科。([GitHub][4])
* 参数支持 `--papers-per-subject`、`--metadata-path`、`--output-dir`、`--model`、`--max-paper-length`、`--chunk-size`、`--chunk-overlap`、`--save-pdfs`、`--random-sample`、`--subjects`、`--delay`、`--summary-only`。这说明它既能跑全学科，也能只挑特定学科，或者只做最终汇总。([GitHub][4])
* `check_subject_availability(...)` 会先读取元数据 CSV，统计每个 subject 下有多少已发表论文，只有存在论文的学科才进入后续处理。([GitHub][4])
* `process_subject(...)` 会拼出一个 shell 命令，去调用 `extract_biorxiv_tasks.py`，把 subject、limit、metadata-path、model、chunk 参数等传进去，再用 `subprocess.run(..., shell=True, check=True)` 执行。([GitHub][4])
* `combine_summaries(...)` 会在每个 subject 的输出目录中读取 `frequency_summary.json`，把其中的 `tasks`、`databases`、`software` 频次汇总到全局，再按频率排序，输出：

  * `combined_summary.json`
  * `tasks_frequency.csv`
  * `databases_frequency.csv`
  * `software_frequency.csv` ([GitHub][4])
* 主程序里还会在不同 subject 之间加 `delay` 秒的等待，默认 `60` 秒，用来降低被限流或压力过大的风险。([GitHub][4])

所以它的定位很明确：

**“全学科批量任务抽取总控脚本”**
相当于把 `extract_biorxiv_tasks.py` 从“单学科工具”提升成“全站主题扫描器”。([GitHub][4])

## 这 3 个脚本之间的关系

这三个文件不是孤立的，而是一个很自然的流水线：

1. `extract_biorxiv_tasks.py`
   负责从一组 bioRxiv 论文中抽取任务、数据库、软件。([GitHub][2])

2. `process_all_subjects.py`
   负责批量对多个学科调用上面这个脚本，并把所有学科结果汇总。([GitHub][4])

3. `generate_function.py`
   负责把抽取出来的任务描述进一步转成 Python 代码。([GitHub][3])

换句话说，这个目录实现的是一种 **“从论文自动归纳方法 → 再自动生成函数原型”** 的半自动工具构建流程。这个结论是根据三个脚本的输入输出关系推断出来的。([GitHub][2])

---

## 一句话总结

* `extract_biorxiv_tasks.py`：单学科论文处理器，下载 bioRxiv PDF 并抽取任务/数据库/软件。([GitHub][2])
* `generate_function.py`：把任务描述批量生成 Python 脚本。([GitHub][3])
* `process_all_subjects.py`：批量遍历所有 bioRxiv 学科，调用抽取脚本并合并全局统计。([GitHub][4])
