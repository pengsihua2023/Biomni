这个 `schema_db` 文件夹里的文件，**不是普通代码文件**，而是一组 `.pkl` 序列化文件。结合 `database.py` 可以看出，它们的作用是给 Biomni 的数据库查询工具提供 **API / 数据库 schema 描述**，让 LLM 先根据 schema 把自然语言请求转成正确的接口调用，再由代码去访问真正的数据库。`database.py` 明确会用 `pickle.load(...)` 读取这些文件；例如 `query_uniprot()` 就会加载 `schema_db/uniprot.pkl`，然后把其中的 schema 放进 system prompt 里，让模型生成 API URL。([GitHub][1])

所以，这个文件夹的整体功能可以概括为：

**“各个外部生物医学数据库/API 的结构说明仓库”**
不是直接存数据，而是存“怎么查这些数据库”的结构化知识。([GitHub][1])

下面按文件逐个说明它们大致对应什么数据库、在 Biomni 里起什么作用。

---

## 整体规律

目录里所有文件都以数据库或资源名命名，比如 `uniprot.pkl`、`reactome.pkl`、`gnomad.pkl`、`clinicaltrials.pkl`。这说明每个 `.pkl` 文件大概率对应一个外部数据库/API 的 schema，用于告诉系统：

* 这个数据库能查什么
* 常见 endpoint / 参数有哪些
* 返回对象大概是什么结构
* 自然语言问题应该怎样映射到接口请求

这一点和 `query_uniprot()` 的实现是吻合的。([GitHub][1])

## 各个文件的功能说明

### 1. `cbioportal.pkl`

对应 **cBioPortal** 的 schema。
主要服务于癌症基因组学查询，比如癌症队列、突变、拷贝数变化、表达、病例和研究项目信息。适合肿瘤相关问题。目录中列出了该文件。([GitHub][2])

### 2. `chembl.pkl`

对应 **ChEMBL** 的 schema。
用于药物化学与生物活性查询，比如小分子、靶点、活性值、assay 和药物发现相关信息。([GitHub][2])

### 3. `clinicaltrials.pkl`

对应 **ClinicalTrials.gov** 的 schema。
用于临床试验检索，例如疾病、干预措施、研究状态、纳入标准、研究阶段等。([GitHub][2])

### 4. `clinvar.pkl`

对应 **ClinVar** 的 schema。
用于临床变异解释，查询某个变异的致病性、相关疾病、提交证据等。([GitHub][2])

### 5. `dailymed.pkl`

对应 **DailyMed** 的 schema。
主要用于药品标签、说明书、适应证、用法用量、警告信息等 FDA 药品说明信息查询。([GitHub][2])

### 6. `dbsnp.pkl`

对应 **dbSNP** 的 schema。
用于单核苷酸变异和短变异位点查询，比如 rsID、位点信息、变异类型等。([GitHub][2])

### 7. `emdb.pkl`

对应 **EMDB (Electron Microscopy Data Bank)** 的 schema。
主要服务于电镜密度图、样品、分辨率和结构条目查询。([GitHub][2])

### 8. `encode.pkl`

对应 **ENCODE** 的 schema。
用于功能基因组学数据查询，例如 ChIP-seq、RNA-seq、ATAC-seq、细胞系、实验条件等。([GitHub][2])

### 9. `ensembl.pkl`

对应 **Ensembl** 的 schema。
用于基因、转录本、蛋白、基因组坐标、同源基因、注释等查询。([GitHub][2])

### 10. `geo.pkl`

对应 **GEO (Gene Expression Omnibus)** 的 schema。
主要用于基因表达和组学实验数据集检索，例如 series、sample、platform 等。([GitHub][2])

### 11. `gnomad.pkl`

对应 **gnomAD** 的 schema。
用于人群变异频率查询，例如变异在人群中的等位基因频率、群体分布、约束性指标等。([GitHub][2])

### 12. `gtopdb.pkl`

对应 **Guide to PHARMACOLOGY / GtoPdb** 的 schema。
主要用于药物、配体、受体、酶、离子通道等药理学靶点信息查询。([GitHub][2])

### 13. `gwas_catalog.pkl`

对应 **GWAS Catalog** 的 schema。
用于疾病/表型与遗传位点关联查询，比如 trait、SNP、研究、关联统计值。([GitHub][2])

### 14. `interpro.pkl`

对应 **InterPro** 的 schema。
用于蛋白家族、结构域、功能位点和注释信息查询。([GitHub][2])

### 15. `iucn.pkl`

对应 **IUCN Red List** 相关 schema。
更偏生态/物种保护信息，用于物种保护状态、分布、威胁等级等查询。([GitHub][2])

### 16. `jaspar.pkl`

对应 **JASPAR** 的 schema。
用于转录因子结合 motif / PWM 数据查询。常见于调控组学和 motif 分析。([GitHub][2])

### 17. `kegg.pkl`

对应 **KEGG** 的 schema。
用于通路、基因、化合物、疾病、模块等 KEGG 资源查询。([GitHub][2])

### 18. `monarch.pkl`

对应 **Monarch Initiative** 的 schema。
用于基因—疾病—表型跨物种整合查询，适合遗传病和表型关联分析。([GitHub][2])

### 19. `mpd.pkl`

对应 **Mouse Phenome Database (MPD)** 的 schema。
用于小鼠品系、表型、实验测量和性状比较查询。([GitHub][2])

### 20. `openfda.pkl`

对应 **openFDA** 的 schema。
用于 FDA 开放数据查询，例如不良事件、药品标签、器械、食品召回等。([GitHub][2])

### 21. `opentarget.pkl`

对应 **Open Targets** 的 schema。
主要用于疾病—基因—药物靶点证据整合查询。适合做靶点发现或疾病机制探索。([GitHub][2])

### 22. `paleobiology.pkl`

对应 **Paleobiology Database** 的 schema。
用于古生物、化石记录、地层和地理分布查询。([GitHub][2])

### 23. `pdb.pkl`

对应 **Protein Data Bank (PDB)** 的 schema。
用于蛋白和大分子三维结构条目、实验方法、链、配体等信息查询。([GitHub][2])

### 24. `pride.pkl`

对应 **PRIDE** 的 schema。
用于蛋白质组学数据集、实验项目、质谱结果等查询。([GitHub][2])

### 25. `pubchem.pkl`

对应 **PubChem** 的 schema。
主要用于化合物、CID、性质、结构、同义词、生物活性等查询。([GitHub][2])

### 26. `quickgo.pkl`

对应 **QuickGO** 的 schema。
用于 Gene Ontology 术语、注释、基因产品与 GO 关系查询。([GitHub][2])

### 27. `reactome.pkl`

对应 **Reactome** 的 schema。
用于生物通路、事件、反应、通路成员和层级关系查询。([GitHub][2])

### 28. `remap.pkl`

对应 **ReMap** 的 schema。
通常用于转录因子结合位点 / ChIP-seq 汇编资源查询。更偏调控组学。([GitHub][2])

### 29. `stringdb.pkl`

对应 **STRING** 的 schema。
用于蛋白—蛋白相互作用网络查询。适合网络生物学和功能关联分析。([GitHub][2])

### 30. `ucsc.pkl`

对应 **UCSC Genome Browser / UCSC API** 的 schema。
用于基因组区间、注释 track、坐标相关资源查询。([GitHub][2])

### 31. `unichem.pkl`

对应 **UniChem** 的 schema。
用于跨化学数据库的化合物 ID 映射。比如 PubChem、ChEMBL 等之间的 ID 对应。([GitHub][2])

### 32. `uniprot.pkl`

对应 **UniProt** 的 schema。
这是目前在 `database.py` 里能直接确认被加载使用的 schema 文件，用于指导 LLM 生成 UniProt REST API 调用。([GitHub][1])

### 33. `worms.pkl`

对应 **WoRMS (World Register of Marine Species)** 的 schema。
用于海洋物种分类、命名和物种信息查询。([GitHub][2])

## 这些文件在系统里的实际作用

从 `database.py` 的实现可以比较可靠地推断出它们的工作流程：

1. 用户给出自然语言问题。
2. 代码读取某个 `.pkl` schema 文件。
3. 把 schema 连同说明一起发给 LLM。
4. LLM 生成结构化 API 请求，例如 URL、参数和查询说明。
5. 代码再用统一的 REST 查询函数真正去访问数据库。([GitHub][1])

所以，这个目录并不是“数据库本身”，也不是“抓回来的数据”，而是 Biomni 的 **数据库接口知识层**。([GitHub][1])

---

## 一句话总结

`schema_db` 文件夹下的每个 `.pkl` 文件，基本都对应一个外部生物医学数据库或 API 的 **schema/接口描述模板**；它们的主要功能是帮助 Biomni 把自然语言问题转换成对这些数据库的正确调用。`uniprot.pkl` 在代码里已经明确这样使用，其余文件按命名和同目录组织方式看，承担的是同类角色。([GitHub][1])
