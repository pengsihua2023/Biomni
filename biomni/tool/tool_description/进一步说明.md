补充两点：

第一，这个目录里的文件大多不是“真正执行分析的代码”，而是 **工具描述清单**。每个文件通常都定义一个统一结构的 `description` 列表，用来告诉系统“有哪些工具、工具名字是什么、参数如何传、工具简介是什么”。这说明它们更像 agent 的工具注册元数据，而不是底层算法实现。

第二，从覆盖范围看，`genomics.py`、`database.py`、`literature.py`、`pharmacology.py` 这几类文件最像 Biomni 的主干能力层，因为它们分别连接了高频科研任务：组学分析、外部数据库访问、文献检索、药物信息查询。其余文件更像按学科扩展出去的专门工具包。这个判断是基于文件内容体量和工具覆盖面的归纳。

你接下来最适合做的一步，是继续看这些 description 文件如何被主程序加载；那样就能把“说明层”和“真正调用层”对应起来。
