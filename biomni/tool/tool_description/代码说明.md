这个文件夹本质上不是“工具实现代码”，而是 **工具注册/说明层**：每个 `.py` 文件里都在定义一个 `description = [...]` 列表，列出该领域下有哪些工具、每个工具做什么、以及它需要哪些必填/可选参数。也就是说，它更像 Biomni 的“工具目录 / schema / 元数据仓库”，而不是算法本体。目录里包含生物化学、基因组学、数据库、文献、免疫学等多个主题文件。([GitHub][1])

我按“文件名 → 主要功能”给你拆开：

* `biochemistry.py`：偏生物化学实验分析，收录了 CD 光谱分析、RNA 二级结构特征提取、蛋白酶动力学、酶动力学 assay、ITC 结合热力学、蛋白保守性分析等工具。([GitHub][2])
* `bioengineering.py`：偏生物工程/神经工程/组织工程，包含细胞迁移分析、CRISPR-Cas9 编辑模拟、钙成像分析、体外释药动力学、肌纤维形态分析、神经活动轨迹解码、whole-cell ODE 模型模拟等。([GitHub][3])
* `bioimaging.py`：偏医学影像与图像处理，主要是 nnUNet 分割、分割可视化、刚性/仿射/可变形配准、批量配准、相似度计算、配准结果可视化。([GitHub][4])
* `biophysics.py`：偏生物物理与形态/动力学定量，包括蛋白无序区预测、细胞形态与骨架定量、组织形变和流动分析。([GitHub][5])
* `cancer_biology.py`：偏肿瘤生物学与癌症组学，含 DDR 网络分析、细胞衰老/凋亡流式分析、体细胞突变检测与注释、结构变异分析、肿瘤表达 NMF 分型、CNV/纯度/倍体/焦点事件分析。([GitHub][6])
* `cell_biology.py`：偏细胞生物学表型定量，包括细胞周期相位显微定量、细胞运动聚类、FACS 分选、免疫表型流式分析、线粒体形态和膜电位分析。([GitHub][7])
* `database.py`：这是一个“外部生物数据库/API 查询中枢”。从文件内容看，它收了大量数据库查询工具，例如 OpenFDA、GWAS Catalog、gnomAD、NCBI BLAST、Reactome、ClinicalTrials.gov、DailyMed 等，重点是把自然语言问题转成数据库检索。([GitHub][8])
* `genetics.py`：偏遗传学与群体/数量遗传分析。已确认的代表工具包括共祖/突变重组模拟并输出 VCF、转录因子结合位点识别、基因组预测线性混合模型；整体上更偏 DNA 序列、遗传变异、群体和育种建模。([GitHub][9])
* `genomics.py`：这是很大的基因组学/单细胞组学文件，包含 scRNA-seq 细胞类型注释、Panhuman/Azimuth 类注释、scVI/scANVI/Harmony/UCE 嵌入、参考映射、ARCHS4 表达查询、基因集富集、Hi-C 染色质互作、比较基因组学、ChIP-seq peak calling、motif enrichment、基因组区间重叠分析、scRNA 标签迁移等。([GitHub][10])
* `glycoengineering.py`：目前比较轻量，主要是糖工程相关基础工具：N-糖基化 motif 扫描、O-糖基化热点启发式预测，以及糖工程外部资源列表。([GitHub][11])
* `immunology.py`：偏免疫实验和免疫表型定量，已确认包含免疫细胞流动条件追踪、CFSE 增殖分析、CD4 T 细胞细胞因子产生分析、EBV 抗体滴度 ELISA、CNS 病灶组织学分析、IHC 图像分析等；整体明显是免疫学实验分析集合。([GitHub][12])
* `lab_automation.py`：偏实验自动化，主要围绕 PyLabRobot，包括脚本测试，以及液体处理/物料处理教程文档提取。([GitHub][13])
* `literature.py`：文献与网页检索层，收录 DOI 补充材料抓取、arXiv、Google Scholar、PubMed、Google 搜索、网页内容提取、PDF 内容提取、高级 web search agent。([GitHub][14])
* `microbiology.py`：偏微生物学与微生物生态建模，包括厌氧消化优化、砷形态分析、菌落计数、细菌基因组注释、CFU 稀释计数、生长动力学、生物膜定量、微生物细胞分割、gLV 群落动力学、RNA 二级结构预测、Gillespie 随机群体模拟。([GitHub][15])
* `molecular_biology.py`：偏分子生物学基础操作，已确认有 ORF 注释、质粒注释、NCBI 基因 CDS 获取、Addgene/NCBI 质粒序列获取，以及短序列/引物比对类工具。([GitHub][16])
* `pathology.py`：偏病理图像与组织定量。已确认的工具包括角膜神经纤维定量、多重组织图像细胞分割与蛋白表达定量、骨 micro-CT 形态计量；从上下文看，这个文件主要就是组织病理和显微图像定量。([GitHub][17])
* `pharmacology.py`：偏药理与药物信息学，文件很大。已确认的功能包括异种移植瘤抑制分析、药物-药物相互作用查询与管理建议、FDA 不良反应查询、FDA 药品标签信息获取；此外文件里也有基于 SMILES 的化学/药学工具。([GitHub][18])
* `physiology.py`：偏生理学/功能影像/时间序列，包含 MRI 面部重建、ABR 波形分析、纤毛摆动频率、蛋白共定位、昼夜节律 cosinor、脑 ADC map、内溶酶体钙动态、脂肪酸 GC 分析、血流动力学、甲状腺激素药代、淀粉样斑块定量。([GitHub][19])
* `protocols.py`：实验方案检索与本地 protocol 文档读取层，包括 protocols.io 搜索、protocol 详情获取、本地 protocol 列表、本地 protocol 文件读取。([GitHub][20])
* `support_tools.py`：通用辅助工具层，包括 Python REPL 执行、读取函数源码、从 Synapse 下载数据。这个文件不对应具体学科，而是给 agent 提供执行与取数能力。([GitHub][21])
* `systems_biology.py`：偏系统生物学建模，包括 FBA、蛋白二聚体网络、代谢网络扰动模拟、蛋白信号网络 ODE 建模、蛋白结构比较、RAS 动力学、以及 DNA 序列问答类 `query_chatnt`。([GitHub][22])

有一个值得注意的点：

* 目录页里列出了 `synthetic_biology.py`，但我直接打开该路径时返回 Not Found，所以这个 fork 当前分支里可能存在目录展示与实际文件状态不一致、文件被移除/改名、或页面索引缓存的问题。我不能可靠地分析这个文件的内容。([GitHub][1])

整体上，这个文件夹可以理解成 Biomni 的 **“学科分区工具注册表”**：

* 一个文件对应一个生物医学子领域；
* 文件内用统一 schema 描述该领域有哪些工具；
* 真正的实现代码大概率在别的模块里，这里主要负责让 agent 知道“有什么工具可用、参数怎么传”。([GitHub][1])
