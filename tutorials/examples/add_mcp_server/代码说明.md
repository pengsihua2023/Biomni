这个文件夹是一个 **“如何给 Biomni 添加 MCP server” 的最小教程示例**，里面只有两个文件：

* `mcp_config.yaml`
* `mcp_example.ipynb` ([github.com](https://github.com/pengsihua2023/Biomni/tree/main/tutorials/examples/add_mcp_server))

它们分工很清楚：

* `mcp_config.yaml` 负责**声明要接入哪些 MCP server，以及把哪些 MCP 工具映射成 Biomni 可调用工具**。([GitHub][1])
* `mcp_example.ipynb` 负责**演示在 Biomni 里实际加载这份配置并使用这些 MCP 工具**。([GitHub][2])

下面分别说。

## 1. `mcp_config.yaml`

这个文件实现的是 **MCP 服务器配置清单**。
它顶层用 `mcp_servers:` 定义了两个 server：`github` 和 `pubmed`。([GitHub][1])

### `github` 这一段的功能

它把 **官方 GitHub MCP server** 接到 Biomni 里，启动方式是 Docker：

* 命令是 `docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN ghcr.io/github/github-mcp-server`
* `enabled: true`
* 描述写的是 “Official GitHub MCP server for repository and issue management”
* 环境变量里把 `GITHUB_PERSONAL_ACCESS_TOKEN` 绑定到 `${GITHUB_TOKEN}`。([GitHub][1])

这说明它的作用是：

**把 GitHub 仓库/issue/项目管理类 MCP 工具整批接入 Biomni**。
而且这类官方 server 走的是“整包接入”的思路，不在 YAML 里逐个列工具。([GitHub][1])

### `pubmed` 这一段的功能

这部分配置了一个 **本地自定义 MCP server**：

* 启动命令是 `python -m biomni.tool.example_mcp_tools.pubmed_mcp`
* `enabled: true`
* 明确列出了要暴露给 Biomni 的两个工具：

  * `search_pubmed`
  * `get_article_abstract` ([GitHub][1])

并且它还给每个工具写了 Biomni 侧的描述和参数 schema：

* `search_pubmed`
  参数有 `query`（必填，字符串）和 `max_results`（可选，默认 10，整数）。([GitHub][1])
* `get_article_abstract`
  参数只有 `pmid`（必填，字符串）。([GitHub][1])

所以这部分的作用是：

**把你自己的本地 MCP server 只挑出必要工具，显式注册进 Biomni。**
这和 GitHub 那种“整包 server”不同，这里演示的是“精细映射工具”的方式。([GitHub][1])

### 这个 YAML 整体想演示什么

它其实在教两种 MCP 接入模式：

* **现成官方 server 接入**：比如 GitHub MCP，直接通过 Docker 跑。([GitHub][1])
* **自定义本地 server 接入**：比如 PubMed MCP，通过 Python 模块启动，并手动声明工具名、描述和参数。([GitHub][1])

## 2. `mcp_example.ipynb`

这个 notebook 实现的是 **实际使用示例**。
开头的 markdown 就写得很直接：
“MCP servers connect external APIs to Biomni. This guide shows how to use the built-in servers and create your own.” ([GitHub][2])

### 第一段代码在做什么

第一段代码做了 4 件事：

1. `import os`
2. `from biomni.agent import A1`
3. 设置 `GITHUB_TOKEN`
4. 创建 Biomni 主代理 `A1(...)`
5. 调用 `agent.add_mcp(config_path="./mcp_config.yaml")` 加载这份 MCP 配置。([GitHub][2])

输出里还能看到：

* `Checking and downloading missing data lake files...`
* `Discovered 80 tools from github MCP server` ([GitHub][2])

这说明 notebook 想证明两件事：

* `A1` 主 agent 支持通过 `add_mcp(...)` 动态加载 MCP server。([GitHub][2])
* GitHub MCP server 加载后，Biomni 能自动发现很多工具，这里示例里发现了 80 个。([GitHub][2])

### 第二段代码/输出在演示什么

第二段不是单纯展示配置，而是在演示 **MCP 工具真的能参与 agent 的任务执行**。
给 agent 的任务是：

> 对一个 `skin_10k.h5ad` 单细胞数据集做完整 end-to-end 分析，并把分析结果和报告推送到 `mcp_gene` 仓库。([GitHub][2])

随后输出里可以看到 agent：

* 先做计划
* 然后写 Python 代码
* 读取 `.h5ad`
* 打印数据维度、obs/var 元数据等
* 后续继续 QC、聚类、注释、差异分析、报告等步骤。([GitHub][2])

这里最关键的示范点不是单细胞分析本身，而是：

**Biomni 可以在一个真实研究任务里，同时利用自身分析能力 + 通过 MCP 接入的外部工具能力。**
从任务描述“push the entire analysis and report to the mcp_gene repository”来看，这就是在展示 GitHub MCP 工具如何并入科研 agent 工作流。([GitHub][2])

## 3. 这两个文件合起来构成的流程

这整个示例文件夹表达的是一条完整流程：

1. 在 `mcp_config.yaml` 中声明外部 MCP server 和工具映射。([GitHub][1])
2. 在 notebook 中实例化 `A1`。([GitHub][2])
3. 调用 `agent.add_mcp(config_path=...)` 让 Biomni 把这些 server 工具吸收进来。([GitHub][2])
4. 然后像普通 Biomni agent 一样发任务，agent 就能把这些新增 MCP 工具纳入执行流程。([GitHub][2])

## 一句话总结

这个文件夹实现的是一个 **MCP 接入教程示例**：

* `mcp_config.yaml`：定义 GitHub 和 PubMed 两类 MCP server 的接入方式，以及 PubMed 工具的参数 schema。([GitHub][1])
* `mcp_example.ipynb`：演示如何在 `A1` agent 中加载这份配置，并在真实分析任务中使用新增的 MCP 工具。([GitHub][2])
