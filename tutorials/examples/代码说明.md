这个 `tutorials/examples` 文件夹是 **Biomni 的示例教程总入口**。当前目录下有 2 个子文件夹和 3 个单独文件：`add_mcp_server/`、`expose_biomni_server/`、`cloning.ipynb`、`plentiCRISPR.txt`、`pylabrobot.ipynb`。([GitHub][1])

各项作用如下。

`add_mcp_server/`
这个子目录是“**把外部 MCP server 接入 Biomni**”的教程示例。它在 examples 目录里被单独列出，说明它是一组独立示例。你前面问过时我已经拆过：它主要演示如何通过配置文件把 GitHub MCP 和自定义 PubMed MCP 工具接进 `A1` agent。([GitHub][1])

`expose_biomni_server/`
这个子目录是反方向的教程：**把 Biomni 自己暴露成 MCP server 给外部客户端调用**。目录中单独列出了这个子文件夹，和 `add_mcp_server/` 形成一对。你前面问过时我也拆过：它包含一个启动 Biomni MCP server 的脚本和一个客户端测试脚本。([GitHub][1])

`cloning.ipynb`
这是一个 **分子克隆 / CRISPR 质粒设计的端到端示例 notebook**。从 notebook 内容可以直接看出，它会实例化 `A1`，然后让 agent 处理一个真实任务：读取本地 `plentiCRISPR.txt` 里的质粒序列，设计针对人类 `B2M` 基因的 sgRNA，并分析 `plentiCRISPR v2` 质粒结构，最后生成适合实验使用的构建结果。notebook 里明确出现了读取 `./data/plentiCRISPR.txt`、调用 `design_knockout_sgrna`、调用 `annotate_plasmid`，以及 `agent.go(...)` 发起整套任务。([GitHub][2])

`plentiCRISPR.txt`
这个文件是给 `cloning.ipynb` 配套使用的 **质粒序列文本数据文件**。虽然目录页只显示文件名，但 `cloning.ipynb` 里明确用 `with open('./data/plentiCRISPR.txt', 'r')` 读取它，并把它当作 `plentiCRISPR v2` 的序列输入，所以它的作用就是作为克隆示例的原始 DNA/质粒序列资源。([GitHub][1])

`pylabrobot.ipynb`
这是一个 **实验室自动化脚本生成示例 notebook**。它同样实例化 `A1`，然后让 agent 回答一个很具体的问题：为 Hamilton STAR backend 生成一段用 PyLabRobot 在 96 孔板中转移液体的脚本。notebook 中先调用 `get_pylabrobot_documentation_liquid()` 抓取 PyLabRobot 液体处理文档，再由 agent 输出一段完整可运行的 Hamilton STAR 异步脚本，包含 deck 布局、tip carrier、plate carrier、吸液和分液步骤。([GitHub][3])

如果把整个目录当成一个教学包来看，它大致分成三类：

* MCP 集成示例：`add_mcp_server/`、`expose_biomni_server/`。([GitHub][1])
* 生物实验设计 / 分子克隆示例：`cloning.ipynb` + `plentiCRISPR.txt`。([GitHub][2])
* 实验室自动化脚本生成示例：`pylabrobot.ipynb`。([GitHub][3])

一句话概括：这个文件夹是在展示 Biomni 的三类典型用法——**接 MCP、把自己变成 MCP、以及直接完成科研任务示例**。([GitHub][1])
