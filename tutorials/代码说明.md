这个 `tutorials/biomni_101.ipynb` 是一个 **Biomni 入门教学 notebook**，作用不是实现某个新算法，而是演示“如何实例化 Biomni agent，并把一个开放式生物医学研究任务交给它完成”。开头的 markdown 直接写了标题 `## Biomni 101`，并说明这是一个 “simple tutorial on the basics of Biomni package”。([GitHub][1])

从 notebook 里能确认的核心流程有三步。

第一步是 **初始化 Biomni 主代理**。
notebook 里先做 `sys.path.append("../")`，然后 `from biomni.agent import A1`，接着实例化 `agent = A1(path="./biomni_data", llm="claude-sonnet-4-5-20250929")`。输出里还能看到它会检查并下载缺失的数据湖文件，这说明这个教程同时在教你如何准备 Biomni 的本地工作目录和模型后端。([GitHub][1])

第二步是 **直接给 agent 一个真实的生物医学研究任务**。
接下来的 markdown 明确说：然后你就可以直接开始向 agent 提出想要的 biomedical research task。示例任务是：
“Plan a CRISPR screen to identify genes that regulate T cell exhaustion, measured by the change in T cell receptor (TCR) signaling between acute (IL-2 only) and chronic (anti-CD3 and IL-2) stimulation conditions. Generate 32 genes that maximize the perturbation effect.”
这说明这个 notebook 的重点是教用户如何把高层研究目标直接交给 Biomni，而不是先手工逐个调用底层工具。([GitHub][1])

第三步是 **展示 agent 会自动调用多种 Biomni 工具完成任务**。
从执行日志能看到，agent 在这个 CRISPR screen 设计任务里会自动导入并使用多个现成工具，包括数据库查询工具 `query_uniprot`、`query_kegg`、`query_reactome`、`query_ensembl`，组学分析工具 `gene_set_enrichment_analysis`，以及分子生物学工具 `design_knockout_sgrna`。这表明教程想展示的不是“聊天回答”，而是 Biomni 会把一个复杂研究问题拆成数据库检索、通路查询、基因筛选和 sgRNA 设计等多个步骤。([GitHub][1])

从 notebook 输出内容还能看出，这个示例任务最后会产出几类研究结果：

* 一个 **32 个基因的 CRISPR screen 候选列表**，并按类别和预期效应整理成表。([GitHub][1])
* 一个 **实验设计说明**，包括 acute 和 chronic 两种刺激条件。([GitHub][1])
* 一份 **实验 protocol / expected outcomes / sgRNA design recommendations**，例如建议每个基因用 3–4 条 sgRNA、加入 non-targeting 和 positive controls、优先打早期 exon。([GitHub][1])

所以，这个 notebook 实际上在演示 Biomni 的一个完整能力链：

**用户给出开放式研究问题 → A1 agent 自动检索相关工具和知识 → 调用数据库与分析函数 → 生成可执行的实验设计建议和候选基因集。** 这也是输出里出现 “Using prompt-based retrieval with the agent’s LLM” 的原因，说明它会先做资源/工具检索，再进入分析。([GitHub][1])

从教学定位上看，`biomni_101.ipynb` 更像一个 **最小上手案例**，重点教你三件事：

1. 怎么创建 `A1` agent。([GitHub][1])
2. 怎么把自然语言科研任务直接交给 Biomni。([GitHub][1])
3. Biomni 会怎样把一个任务自动拆解成数据库查询、基因集分析和实验设计输出。([GitHub][1])
